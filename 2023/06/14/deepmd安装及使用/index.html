deepmd安装及使用

Created2023-06-14Updated2023-06-14

离线安装

  1. 离线包下载
  2. 将两个文件合并为一个
    • cat deepmd-kit-2.2.2-cuda11.6_gpu-Linux-x86_64.sh.0 deepmd-kit-2.2.2-cuda11.6_gpu-Linux-x86_64.sh.1 > deepmd-kit-2.2.2-cuda11.6_gpu-Linux-x86_64.sh
  3. sh deepmd-kit-2.2.2-cuda11.6_gpu-Linux-x86_64.sh
    • Enter
    • yes
  4. 测试是否安装成功
    • /data/home/scv5029/deepmd-kit/bin/dp -h
    • 其中/data/home/scv5029/deepmd-kit为安装路径

      教程

  5. 训练过程
    • 数据 –>训练–>冻结/压缩模型
    • 准备数据是将DFT的计算结果转换为DeePMD套件可以识别的数据。
    • 训练是使用 DeePMD 套件使用上一步中准备的数据训练深度潜力模型。
    • 将训练过程中的重启文件冻结/压缩到模型中。
  6. 教程数据
    • 准备数据00.data
      • 假设您有一个用于分子动力学的“OUTCAR”，其中包含 1000 个帧。set_size=200表示这1000个点将被分成5个子集，分别命名为data/set.000~data/set.004。每组大小为200。在这5组中，data/set.000-data/set.003将被DeePMD-kit视为训练集，并被data/set.004视为测试集。默认情况下，最后一组将被DeePMD-kit视为测试集。如果只有一组，则该组既是训练集又是测试集。（当然，这样的测试集是没有意义的。）
      • 我们提供的“OUTCAR”只包含1个框架，所以在“data”目录下（与“OUTCAR”在同一目录下）只有1组：data/set.000. 如果要使用这些数据，需要完成一些程序。现在我们只是跳过这些细节并使用我们为您准备的数据。数据位于我们教程数据的根目录中。
    • 训练01.train
      • input.json参数
        • DeePMD-kit的强大之处在于相同的训练参数适用于不同的系统，所以我们只需要稍微修改一下input.json就可以开始训练了。
        • 开始训练dp train input.json
        • 在训练过程中，我们可以看到lcurve.out观察到误差减少。其中，第4列和第5列是能量的测试和训练误差（按原子数归一化），第6和7列是能量的测试和训练误差力。
    • 冻结/压缩模型
      • 冻结dp freeze -o graph.pb
      • 输出模型的默认文件名是graph.pb. 我们可以使用以下脚本来压缩模型：dp compress -i graph.pb -o graph-compress.pb
  7. 工作流程
    • 数据准备：基于从头计算生成训练数据并进行格式转换;
    • 模型训练：准备输入脚本，然后使用数据和脚本训练DP模型;
    • 模型应用：使用获得的DP模型进行MD仿真或模型推理。
  8. 示例
    • 详见气相甲烷分子
    • AIMD数据生成
      • DeePMD-kit的训练数据包含原子类型、模拟框、原子坐标、原子力、系统能量和维里。具有此信息的分子系统的快照称为frame。一个数据系统包括许多共享相同数量原子和原子类型的frame。例如，分子动力学轨迹可以转换为数据系统，每个时间步长对应于系统中的一个frame
    • training_data
      • type.raw
      • type_map.raw
        • 有时用户需要将整数类型映射到原子名称。映射可以由type_map.raw文件给出
  9. 运行教程
    • 数据集：Si216
    • 数据格式转换
      • DeePMD-kit 采用压缩数据格式。所有的训练数据都应该首先转换成这种格式，然后才能被 DeePMD-kit 使用。
      • pip install dpdata，使用dpdata进行数据格式转换可以分两步完成：load data和dump data。
      • dpdata提供将vasprun.xml、POSCAR、OUTCAR等格式文件转换成.npy格式。
      • 按顺序选择前800frames作为训练数据，其余作为验证数据
      • 已经完成数据转换。DeePMD-kit 格式的数据存储在文件夹 00.data 中
      • training_data
    • 输入脚本
      • 新建训练目录01.train，包含input.json训练脚本。通过在input.json中指定关键字的值来指定训练过程。具体参考关键字解释手册
      • 复制CH4中的input.json，此处对该文件做如下修改，参考
        • 原子名称"type_map":["Si"],
        • 数据写入的文件路径"systems": ["../00.data/training_data"],
        • "numb_steps":1000000,
        • "decay_steps":5000,其中"decay_steps=numb_steps/200"
        • "sel": [216],数据集仅包含一种类型的元素（“Si”），将其更改为"sel": [216],。否则会报错ValueError: Cannot feed value of shape (3,) for Tensor d_sea_t_natoms:0, which has shape (4,)
    • 训练过程dp train input.json
      • 可以看到数据系统的信息以及本次训练的起始和最终学习率。每1000步打印一次信息
      • DEEPMD INFO start training at lr 1.00e-03 (== 1.00e-03), decay_step 5000, decay_rate 0.950006, final lr will be 3.51e-08
      • 在第 10000 个批次结束时，模型保存在Tensorflow的检查点文件中model.ckpt。同时，训练和测试错误在lcurve.out文件中
      • cat lcurve.out查看lcurve.out
        • lcurve.out 包含 8 列，从左到右分别是训练步骤、验证损失、训练损失、均方根（RMS）能量验证误差、RMS 能量训练误差、RMS 力验证误差、RMS力的训练误差和学习率。能量的均方根误差 (RMSE) 通过系统中的原子数归一化。
      • 可视化lcurve.out
      • 当训练过程异常停止时，我们可以通过简单地运行dp train --restart model.ckpt input.json从提供的检查点重新开始训练
      • 冻结模型dp freeze -o graph.pb
        • 其中 -o 表示输出，用于指定输出模型的名称
        • 输出一个以graph.pb当前目录命名的模型文件
        • 压缩模型压缩的 DP 模型通常可以将基于 DP 的计算速度提高一个数量级，并且消耗更少的内存。
      • 压缩模型dp compress -i graph.pb -o graph-compress.pb
        • 其中 -i 表示输入，用于导入未冻结的模型
        • 输出一个名为graph-compress.pb
      • 测试模型dp test -m graph-compress.pb -s ../00.data/validation_data -n 40 -d results
        • 其中-m表示model，用于导入模型文件，-s表示system，指定测试数据集的路径，-n表示number，指定要测试的帧数，-d表示detail，写入细节能量、力和力力到不同的文件。
        • 在当前目录中输出名为results.e.out和results.f.out的文件
    • 模型应用
      • 使用 LAMMPS 运行 MD，创建02.lmp文件夹
      • 将训练目录中的输出模型软链接到当前目录
        • ln -s ../01.train/graph-compress.pb
        • 该文件夹出现三个文件conf.lmp、graph-compress.pb、in.lammps
          • 其中conf.lmp给出了气相甲烷 MD 模拟的初始配置，in.lammps是 LAMMPS 输入脚本
      • 运行lammpslmp -i in.lammps
      • 生成log.lammps和ch4.dump文件。它们分别存储热力学信息和分子的轨迹。
    • 模型推理
      • 可以使用 DP 模型来评估 LAMMPS 轨迹中每一frame的能量
